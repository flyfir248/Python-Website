Object Oriented Programming

Object-oriented programming (OOP) is a programming paradigm that uses objects, which are instances of classes, to represent and manipulate data and functionality.

OOP is based on several key principles:

  • Classes: Classes are templates or blueprints that define the structure and behavior of objects. They encapsulate data (attributes) and methods (functions) that operate on the data. Classes serve as a blueprint for creating objects.
  • Objects: Objects are instances of classes. They are actual data structures created based on a class definition. Objects contain both data (attributes or properties) and methods (functions) that can interact with and manipulate that data.
  • Encapsulation: Encapsulation is the principle of bundling data and the methods that operate on that data within a single unit, which is the class. It helps hide the internal details of an object and only exposes a well-defined interface. This provides data protection and ensures that the object's internal state cannot be easily modified from outside the object.
  • Inheritance: Inheritance allows one class (the subclass or derived class) to inherit properties and methods from another class (the superclass or base class). This promotes code reusability and the creation of hierarchical relationships between classes.
  • Polymorphism: Polymorphism means that objects of different classes can be treated as objects of a common base class. This allows for more flexible and generic code, as different objects can respond to the same method calls in a way that is appropriate for their specific class.
  • Abstraction: Abstraction involves simplifying complex reality by modeling classes based on the essential properties and behaviors, while ignoring or hiding the unimportant details. It allows programmers to work with high-level, understandable concepts rather than low-level implementation details.

OOP promotes modularity, code reusability, and a more intuitive representation of real-world objects and their interactions, making it a popular choice for software development in various domains. Common object-oriented programming languages include Java, C++, Python, and C#.

What is a Class?

Template/Blueprint/Mould of any object, which may contain some initial state (value/variables) and behaviors/actions (functions).

What is an Object?

An instance/object of a class.

What is Instantiation?

Process of creating an object.

How do we do Instantiation?

object_name = ClassName()

What is a Constructor?

It's a function that is executed during the creation of an object. It is written as def __init__(self):

Steps for Creating an Object:

  1. Define the class syntax:
    2.                 
        class ClassName:
            def __init__(self):
                # body of __init__

            def function1(self):
                # body of function (optional)
  2. Execute the class.
  3. Instantiate objects.
  4. Call functions or access variables.

Steps of Execution Happening at Execution:

  1. Execute the class: Class will be loaded into memory. No object is created, but the class/instantiate will be available to create objects. Without executing the class, we cannot instantiate the object.
  2. Instantiate:
    • __init__ is executed.
    • All the instance variables defined in __init__ are created.
  3. Calling function:
    • Instance variables inside the function are created or modified.

        class Electric(Vehicle):   # ----- Electric is child class Vehicle is Parent class
            def __init__(self, br, mdl, yr, veh_num, pr, kpc):
                self.vehicle_type = 'Electric'
                self.km_per_charge = kpc
                super().__init__(br, mdl, yr, veh_num, pr)

            def sell(self, dep_val=0.3):
                self.owner_number = self.owner_number + 1
                self.price = self.price - self.price * dep_val
                print('Electric vehicle sold')
